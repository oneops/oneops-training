# Traditional Cloud Application

- Defined via 
  - Provisioning tools
  - Configuration mgt tools
  - Custom scripts
- Specific deployment
  - Cloud API usage


# OneOps Assembly

An application defined in OneOps including

- Components 
- Monitoring
- Cloud requirements
- Sizing and scaling
- And more

_Abstracted_ from specific tools and platforms.


# _Cloud_ in OneOps 

__Abstract__ set of services

- Compute
- DNS
- Load balancing
- ...


#  

## OneOps assembly 

is deployed to 

## Cloud(s) in OneOps


# Benefits

Assembly is __independent__ of deployment cloud:

- different cloud providers
- mix and match vendors and services
- changes over time
- hybrid private and public cloud
- assembly dev != OneOps cloud admin


# Cloud-Related Features

* Scaling
* Auto-healing
* Monitoring
* Reporting
* Hybrid clouds
* Seamless migration
* Policy auditing

> And much more...
